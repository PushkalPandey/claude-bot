🔀 handleMessage() — What kind of message is it? main.go:165
⛔ "stop" or "kill"
Calls cancelFunc() which cancels the running Claude process's context. The subprocess is killed. Replies "⛔ Killed." in thread.
🔄 "reset" or "new session"
Deletes the stored session ID for this Slack channel. The next message will start a brand-new Claude conversation. Replies "🔄 Session reset." in thread.
✅ Any other text
Treated as a prompt for Claude. Checks if Claude is already running — if so, rejects with a warning. Otherwise, proceeds to execute Claude CLI.

⚙️ Build & run Claude CLI command main.go:229–235
Posts a 🤔 Thinking… placeholder message in the thread, then builds the command:

claude --dangerously-skip-permissions --verbose -p "<prompt>" --output-format stream-json

If this channel has a previous session ID saved, appends --resume <session_id> so Claude remembers the conversation history. The command runs with a cancellable context so it can be killed mid-run by the "stop" command. stdout is piped for live reading.

📡 Stream-JSON reader goroutine main.go:266–309
A background goroutine scans stdout line by line. Each line is a streamEvent JSON object:

• type "assistant" — accumulates text; feeds each tool_use block through describeToolUse() to produce an emoji + label step (e.g. 🖥️ Bash: `go build …`).
• type "result" — stores the final session_id, is_error, duration_ms, total_cost_usd, and num_turns.

All writes go through a sync.Mutex shared with the ticker goroutine.
ticker fires every 2 s
🔄 updateMessage() — live Slack edit every 2 s main.go:360–418
A time.Ticker fires every 2 seconds and calls updateMessage() to edit the placeholder in place. The live message shows:

• ⏱ elapsed time & token counts
• Steps taken: tool-use log (last 10 steps)
• Response (streaming…): last 1 800 chars of accumulated text

When the reader goroutine closes (Claude exits), the ticker stops, cmd.Wait() returns, and a final updateMessage() is sent with the complete response plus a ✅ Done · Xs · N turns · $0.00XX cost footer. Responses over 3 900 chars are truncated.
Global State (shared memory)
runningCmd
*exec.Cmd
Holds the currently-running Claude subprocess. Used to detect if Claude is already busy. Set to nil when the process finishes.
runningMu
sync.Mutex
Protects runningCmd and cancelFunc from race conditions. Because multiple Goroutines can handle messages at the same time, this lock ensures only one goroutine modifies these values at a time.
cancelFunc
context.CancelFunc
A function that, when called, cancels the running Claude command's context — killing the subprocess. The "stop"/"kill" command calls this.
sessions
map[string]string
Maps Slack channel ID → Claude session ID. Allows each channel to have its own continuous conversation with Claude. Cleared by the "reset" command.
sessionsMu
sync.Mutex
Protects the sessions map from concurrent read/write.

// Top-level event — one per stdout line
type streamEvent struct {
    Type        string        `json:"type"`            // "assistant" | "result" | …
    Message     *assistantMsg `json:"message,omitempty"`
    Result      string        `json:"result,omitempty"`   // final text (result event)
    SessionID   string        `json:"session_id,omitempty"`
    IsError     bool          `json:"is_error,omitempty"`
    DurationMs  int64         `json:"duration_ms,omitempty"`
    TotalCostUSD float64      `json:"total_cost_usd,omitempty"`
    Usage       *tokenUsage   `json:"usage,omitempty"`
    NumTurns    int           `json:"num_turns,omitempty"`
}

// Nested inside streamEvent.Message
type assistantMsg struct {
    Content []contentBlock `json:"content"`
    Usage   *tokenUsage   `json:"usage,omitempty"`
}

// One text or tool_use block inside an assistant message
type contentBlock struct {
    Type  string          `json:"type"`    // "text" | "tool_use"
    Text  string          `json:"text,omitempty"`
    ID    string          `json:"id,omitempty"`
    Name  string          `json:"name,omitempty"`   // tool name
    Input json.RawMessage `json:"input,omitempty"`
}

// toolStep — emoji+label pair shown in the live Slack feed
type toolStep struct { emoji, label string }

Concurrency Model
🔒 Mutex locks
Two mutexes protect shared state: runningMu protects the active subprocess and cancel function, sessionsMu protects the session map. This prevents data races when two messages arrive at the same time.
⚡ Goroutines
Each incoming message is handled in its own goroutine (go handleMessage(...)). Inside handleMessage, a second goroutine reads the stream-json stdout pipe, while a time.Ticker (every 2 s) edits the Slack placeholder with live progress. The event-listening loop and the main thread both run independently.
❌ Only one Claude at a time
Even though messages are handled concurrently, the bot enforces that only one Claude subprocess runs at a time (via runningCmd check). New prompts are rejected with a warning until the current one finishes or is killed.
🎯 Context cancellation
Claude runs inside a context.WithCancel. When "stop" is sent, cancelFunc() is called which propagates a cancellation signal to the subprocess, terminating it cleanly.
